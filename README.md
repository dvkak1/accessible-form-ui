# Accessible Form UI — roadmap.sh Challenge

A frontend project focused on building an **accessible, user-friendly form interface** using semantic HTML and CSS. This project is part of the roadmap.sh frontend practice series, which emphasizes real-world UI patterns and usability.

---

## 📌 Overview

The **Accessible Form UI** project focuses on designing a form that is not only visually clean but also **usable by all users**, including those relying on keyboard navigation and assistive technologies.

Rather than just styling inputs, this project emphasizes:

* Semantic structure
* Accessibility (a11y)
* User experience (UX)
* Error handling clarity

---

## 🎯 Objectives

* Build a fully structured and accessible form layout
* Apply proper **label-input relationships**
* Ensure **keyboard navigability**
* Provide meaningful **error feedback**
* Follow accessibility best practices (ARIA where necessary)

---

## 🧱 Features

* Properly associated `<label>` and `<input>` elements
* Input types optimized for usability (`email`, `password`, etc.)
* Accessible error messaging with descriptive feedback
* Logical tab navigation (keyboard-friendly)
* Clear visual hierarchy and spacing
* Form validation-ready structure

---

## 🛠️ Built With

* HTML5 (semantic elements)
* CSS3 (layout, spacing, visual feedback)
